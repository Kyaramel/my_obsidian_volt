# アプリの基本構成
チャットベースのアプリ「長岡Reco麺der」を作成。
- 画面上部に「長岡Reco麺der」と表示（フォント: 太字、色: #1E90FF、サイズ: 1.5emでデバイスに応じて調整）。
- 中央に白背景のチャットエリアを配置し、ユーザー入力（例: "醤油、濃い"）とAI応答（例: "おすすめ: 柿川亭 - 醤油/なし/濃い"）を対話形式で表示。
- 入力欄は画面下部にテキストボックスと「送信」ボタン（色: #32CD32、サイズ: 1em）を配置。
- レスポンシブデザインでモバイル対応。

# 機能仕様
- **入力処理**
    - ユーザーがチャットでラーメンの好みを入力（例: "醤油、濃い"）すると、CSVファイル 'ramen.csv' を読み込み。
- **キーワード抽出**
    - 入力文をカンマまたはスペースで分割し、複数のキーワード（例: "醤油", "濃い"）を抽出。
- **マッチングロジック**
    - 各行の「味」「スープ」「あっさり/濃い」列からキーワードを抽出し、入力の各キーワードとマッチする数をカウント。
    - 漢字（例: 醤油）、ひらがな（例: しょうゆ）、カタカナ（例: ショウユ）の表記揺れを無視し、部分一致または意味的に近いキーワードとしてマッチング。
        - 例: "しょうゆ" → "醤油" や "醤項とんこつ" にマッチ。
        - 例: "こい" → "濃い" にマッチ。
- **推薦結果**
    - マッチ数が最も多い上位1-3店を推薦。マッチ数が同数の場合、複数表示。
- **結果表示**
    - 結果に店名と特徴（味 + スープ + あっさり/濃いを結合）を表示。
- **エラー処理**
    - 曖昧な入力（例: "濃いめ"）には「濃いかあっさりか教えてね」と返答。

# 実装要件
- **CSV設定**
    - CSV 'ramen.csv' は UTF-8 BOM付きでアップロード（例: 柿川亭, 醤油, なし, 濃い）。
- **フォントサイズ**
    - フォントサイズは相対単位（em）を使用し、デバイスの画面サイズに自動調整。最小16px、最大24pxを目安。
- **UIデザイン**
    - チャットUIはシンプルに、ユーザー入力とAI応答を時系列で表示。
